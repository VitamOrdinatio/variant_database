# Plans Namespace

Namespace: `docs/plans/`

**Plans = Future execution sequencing**

> Core question:
> What work should be performed to implement VDB?

Examples:

- implementation plans
- phased development plans
- DEX-facing execution plans
- feature rollout plans

Plans documents answer:

1. What should be built?
2. In what sequence should work occur?
3. What implementation tasks are required?
4. What work remains future-facing?

Plans are:

- forward-looking
- execution-oriented
- implementation-adjacent
- time-sensitive

> Principle:
> Plans define future implementation sequencing.

> Anti-pattern:
> Plans should not redefine architecture,
> replace contracts, or document completed work.
