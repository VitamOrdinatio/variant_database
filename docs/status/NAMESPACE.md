# Status Namespace

Namespace: `docs/status/`

**Status = Current repository state**

> Core question:
> What is currently true about VDB maturity?

Examples:

- phase status reports
- implementation checkpoints
- readiness summaries
- current limitation notes

Status documents answer:

1. What is currently implemented?
2. What is currently incomplete?
3. What has recently changed?
4. What is the current maturity state?

Status is:

- current-state-focused
- checkpoint-oriented
- time-sensitive
- operationally descriptive

> Principle:
> Status documents communicate current repository maturity and progress.

> Anti-pattern:
> Status documents should not become release histories
> or implementation plans.
