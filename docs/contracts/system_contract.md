# system_contract.md

## Purpose

This document defines the top-level system contract for the Variant Database (VDB).

The system contract establishes the non-negotiable obligations that all VDB implementation work must satisfy.

This contract governs VDB as a preservation-first evidence system.

---

# Scope

This contract applies to all VDB implementation domains, including:

```text
ingestion
persistence
namespace governance
discovery
query surfaces
validation
external evidence capsules
RDGP-facing export surfaces
future producer integration
```

This contract does not define implementation details for any single subsystem.

Subsystem-specific obligations are delegated to mini-contracts.

---

# System Role

VDB is the persistent evidence steward for the VitamOrdinatio repository ecosystem.

VDB must preserve, relate, expose, and reconstruct evidence generated by producer repositories.

Initial producer families include:

```text
VAP
GSC
```

Planned or future producer / consumer families include:

```text
RSP
RDGP
TEP-VDB
TEP-RDGP
external evidence capsules
```

---

# Core System Principle

```text
VDB preserves evidence before it simplifies evidence.
```

Every implementation decision must protect:

```text
identity
provenance
authority
uncertainty
evidence topology
future reinterpretability
reconstructability
```

---

# System Boundary

VDB is responsible for:

```text
reading certified producer TEPs
validating ingestion readiness
registering metadata
preserving artifacts and evidence states
brokering namespaces
attaching overlays
supporting discovery
constructing query surfaces
emitting RDGP-ready surfaces
supporting evidence reconstruction
```

VDB is not responsible for:

```text
modifying producer artifacts
rerunning producer pipelines
overwriting producer evidence
performing clinical diagnosis
performing RDGP reasoning
declaring biological causality
```

---

# Producer Artifact Immutability

VDB must treat producer artifacts as immutable.

For canonical producer evidence:

```text
VDB may read producer artifacts.

VDB may register producer artifacts.

VDB may preserve references to producer artifacts.

VDB must not mutate producer artifacts.
```

This applies especially to certified producer TEPs.

---

# Evidence Preservation Obligations

VDB must preserve the following as first-class logical domains:

```text
TEP Package
Artifact
Entity
Evidence Object
Evidence State
Evidence State Transition
Provenance Event
Namespace Event
Overlay Attachment
Discovery Event
Query Surface
```

VDB must not reduce these domains into a single undifferentiated table or score.

---

# Evidence Object Contract

VDB must represent evidence-bearing units as Evidence Objects.

An Evidence Object must preserve:

```text
source identity
producer identity
source artifact references
evidence family
evidence domain
authority class
provenance references
```

Evidence Objects may participate in multiple Evidence States over time.

---

# Evidence State Contract

VDB must represent interpreted or transported evidence conditions as Evidence States.

Evidence States must preserve:

```text
state identity
state type
source Evidence Object
producer context
generation context
authority context
uncertainty context
```

Evidence States must be additive and reconstructable.

---

# Evidence State Transition Contract

VDB must represent state changes as Evidence State Transitions.

Examples include:

```text
producer observation to interpreted state
ingested state to persisted state
persisted state to overlay-attached state
query-surface projection state
RDGP-returned reasoning state
```

State transitions must not overwrite prior states.

---

# Metadata Obligations

VDB must register metadata for:

```text
producer
repository
run
release
TEP package
artifact
entity
evidence object
evidence state
external evidence capsule
query surface
```

Metadata must preserve source identity and authority.

---

# Provenance Obligations

VDB must preserve provenance sufficient to reconstruct:

```text
where evidence came from
which artifact contained it
which producer emitted it
which run generated it
which TEP transported it
which transformations affected it
which overlays attached to it
which query surfaces exposed it
```

Evidence without reconstructable provenance is not VDB-compliant.

---

# Namespace Governance Obligations

VDB must preserve source identities before attaching canonical identities.

Namespace resolution may enrich evidence.

Namespace resolution must not replace source identity.

Required namespace behaviors:

```text
source identifiers retained
source namespaces retained
canonical identifiers attached separately
resolution status visible
resolution provenance visible
ambiguity preserved
conflicts preserved
```

Resolution is optional.

Preservation is mandatory.

---

# Discovery Obligations

VDB may discover additional evidence context.

Examples include:

```text
internal evidence relationships
cross-producer overlays
external metadata
ontology context
future reinterpretation opportunities
```

Discovery must preserve:

```text
discovery event
authority source
retrieval timestamp
field-level provenance
attachment rationale
conflict status
```

Discovery must not silently enrich or rewrite producer evidence.

---

# Overlay Attachment Obligations

VDB may attach evidence from one domain to evidence from another domain.

Examples include:

```text
GSC phenotype-gene overlays
future RSP transcriptomic overlays
external metadata overlays
future RDGP reasoning overlays
```

Overlay attachment must preserve:

```text
primary evidence identity
overlay evidence identity
attachment basis
attachment policy
attachment provenance
attachment uncertainty
phenotype context when applicable
```

Overlay attachment must not fuse evidence.

---

# Query Surface Obligations

VDB must expose governed query surfaces rather than requiring consumers to query persistence directly.

Query surfaces must preserve:

```text
identity
provenance
authority
uncertainty
evidence channels
overlay participation
source lineage
```

Query surfaces are derived access surfaces.

Query surfaces are not authoritative replacements for preserved evidence.

---

# RDGP Surface Obligations

VDB must support construction of RDGP-facing evidence surfaces.

RDGP surfaces must be:

```text
sample-gene centered
deterministic
provenance-complete
namespace-aware
uncertainty-aware
overlay-aware
return-path-ready
```

VDB prepares evidence for RDGP.

RDGP reasons over evidence.

---

# External Evidence Capsule Obligations

VDB may ingest or attach externally discovered evidence through External Evidence Capsules.

External Evidence Capsules must preserve:

```text
external authority
retrieval event
retrieval timestamp
snapshot identity
field-level provenance
attachment status
conflict status
```

External evidence must remain distinguishable from producer evidence.

---

# Evidence Reconstruction Obligations

VDB must support reconstruction of preserved evidence.

Given an Evidence Object, Evidence State, artifact, TEP, overlay, or query-surface record, VDB must be able to reconstruct:

```text
source repository
source run
source TEP
source artifact
source entity
source evidence object
state history
namespace history
overlay history
discovery history
query-surface derivation
```

If evidence cannot be reconstructed, preservation has failed.

---

# Validation Obligations

All implementation must satisfy validation doctrine.

Implementation must support:

```text
schema validation
ingestion validation
namespace resolution validation
lifecycle validation
evidence reconstruction validation
```

Validation must confirm preservation, not biological correctness.

---

# Determinism Obligations

VDB outputs must be deterministic under fixed inputs.

Given the same:

```text
source evidence
schema versions
namespace policies
aggregation policies
overlay policies
query-surface versions
```

VDB must produce equivalent outputs.

---

# Anti-Collapse Rules

The following are prohibited:

```text
identity collapse
provenance collapse
authority collapse
namespace collapse
overlay fusion
state overwrite
query-surface authority inversion
external metadata silent enrichment
RDGP reasoning overwrite
```

Any implementation that performs one of these actions violates this system contract.

# Compliance Rule

No implementation phase may be considered complete unless it satisfies this system contract and all relevant mini-contracts.

Implementation convenience must not override preservation obligations.

---

# Summary

The VDB system contract exists to protect the core mission of the repository.

VDB must preserve evidence such that future systems can identify, trust, relate, query, and reconstruct it.

The guiding rule is:

```text
Preserve broadly.

Expose narrowly.

Never collapse the evidence.
```
