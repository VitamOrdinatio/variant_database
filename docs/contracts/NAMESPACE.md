# Contracts Namespace

Namespace: `docs/contracts/`

**Contracts = Guarantees and invariants**

> Core question:
> What must VDB preserve, enforce, and guarantee?

Examples:

- ingestion contracts
- persistence contracts
- namespace governance contracts
- query surface contracts

Contract documents answer:

1. What must always be true?
2. What guarantees are provided?
3. What invariants must be preserved?
4. What behaviors are considered contractually required?

Contracts are:

- enforceable
- implementation-independent
- stable
- governance-oriented

> Principle:
> Contracts define guarantees independent of implementation.

> Anti-pattern:
> Contracts should not prescribe how guarantees are implemented.
