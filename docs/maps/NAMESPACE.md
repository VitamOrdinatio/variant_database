# Maps Namespace

Namespace: `docs/maps/`

**Maps = Strategic directionality**

> Core question:
> What milestones define VDB's intended maturation?

Examples:

- milestone_map.md
- roadmap maps
- strategic feature maps
- ecosystem maturation maps

Maps documents answer:

1. Where is VDB going?
2. What milestones define repo maturation?
3. What strategic capabilities matter?
4. How does the repo evolve over time?

Maps are:

- strategic
- directional
- milestone-oriented
- high-level

> Principle:
> Maps define strategic repository direction and long-term evolution.

> Anti-pattern:
> Maps should not become implementation plans,
> contracts, or subsystem designs.
