# Examples Namespace

Namespace: `docs/examples/`

**Examples = Concrete artifacts**

> Core question:
> What small examples make VDB behavior inspectable?

Examples:

- toy manifests
- toy discovery reports
- toy ingestion examples
- toy query outputs

Example documents answer:

1. What does this concept look like concretely?
2. How can a reviewer inspect expected behavior?
3. What minimal artifact demonstrates the idea?
4. How does an abstract rule appear in practice?

Examples are:

- concrete
- illustrative
- reviewer-friendly
- non-authoritative

> Principle:
> Examples make abstract VDB concepts concrete and inspectable.

> Anti-pattern:
> Examples should not become canonical specifications,
> contracts, or architectural authorities.
