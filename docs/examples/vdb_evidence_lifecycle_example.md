# VDB Evidence Lifecycle Example

## HG002 POLG Evidence Lifecycle Walkthrough

### Purpose

This document provides a worked example of the VDB evidence lifecycle using real producer artifacts.

Unlike validation walkthroughs, which verify architectural correctness, this document serves as an illustrative example of how evidence is expected to move through the VDB ecosystem.

The example traces a single evidence object associated with the gene POLG as it moves from:

* VAP
* into VDB
* through namespace resolution
* through discovery overlay attachment
* onto the RDGP query surface

The walkthrough uses real certified artifacts available during VDB design and serves as a benchmark hypothesis for future implementation validation.

# Expected Architectural Outcome

At the conclusion of the lifecycle:

```text
TEP-VAP
    ↓

Evidence Object

    ↓

Namespace Resolution

    ↓

Discovery Overlay Attachment

    ↓

Evidence State Transition

    ↓

Provenance Preservation

    ↓

RDGP Query Surface
```

The original producer artifacts remain preserved.

No evidence collapse occurs.

No provenance is lost.

Cross-producer discovery becomes possible without sacrificing producer identity.

This represents the intended operational behavior of the VDB architecture.
