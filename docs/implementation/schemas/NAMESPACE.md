# Schemas Namespace

Namespace: `docs/implementation/schemas/`

**Schemas = Exact structural definitions**

> Core question:
> What fields, tables, records, and datatypes exist?

Examples:

- data_schema.md
- discovery_schema.md
- metadata_schema.md
- provenance_schema.md
- rdgp_query_surface_schema.md
- relational_schema.md

Schemas documents answer:

1. What structures exist?
2. What fields exist?
3. What datatypes are expected?
4. How are entities represented?

Schemas are:

- structural
- precise
- implementation-facing
- deterministic

> Principle:
> Schemas define exact structures that implementation must produce,
> store, or expose.

> Anti-pattern:
> Schema documents should not explain system philosophy,
> subsystem strategy, or cross-repository responsibilities.
