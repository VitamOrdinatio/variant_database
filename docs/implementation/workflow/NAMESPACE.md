# Workflow Namespace

Namespace: `docs/implementation/workflow/`

**Workflow = Operational sequence**

> Core question:
> What steps occur from discovery through ingestion and query?

Examples:

- workflow.md

Workflow documents answer:

1. What sequence of operations occurs?
2. What transitions exist between stages?
3. What dependencies exist between activities?
4. How does information move through VDB?

Workflow is:

- sequential
- operational
- implementation-facing
- process-oriented

> Principle:
> Workflow documents define realized operational sequences.

> Anti-pattern:
> Workflow documents should not redefine architecture,
> invent new subsystem responsibilities, or replace implementation plans.
