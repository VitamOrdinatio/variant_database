# Implementation Namespace

Namespace: `docs/implementation/`

**Implementation = Realized operational behavior**

> Core question:
> How are architecture, design, interfaces, and contracts made operational?

Examples:

- schemas/
- specifications/
- workflow/
- queries/

Implementation documents answer:

1. How does VDB operate in practice?
2. How are guarantees realized?
3. How are architectural concepts represented operationally?
4. What concrete structures and behaviors exist?

Implementation is:

- operational
- concrete
- execution-oriented
- architecture-derived

> Principle:
> Implementation realizes architecture, design, interfaces, and contracts.

> Anti-pattern:
> Implementation documents should not redefine architecture,
> change interface responsibilities, or alter contractual guarantees.
