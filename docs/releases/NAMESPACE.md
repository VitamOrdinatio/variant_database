# Releases Namespace

Namespace: `docs/releases/`

**Releases = Historical milestones**

> Core question:
> What changed at release checkpoints?

Examples:

- release notes
- version summaries
- release readiness summaries
- milestone closure documents

Release documents answer:

1. What was released?
2. What changed at a release point?
3. What capabilities were present?
4. What limitations were known at release time?

Releases are:

- historical
- milestone-oriented
- version-aware
- stable after publication

> Principle:
> Release documents preserve historical repository milestones.

> Anti-pattern:
> Release documents should not describe future work
> or current repository status.
